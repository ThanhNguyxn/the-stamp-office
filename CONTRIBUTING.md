## Conventional Commits
Use one of these prefixes:
- `feat:` new feature (later, when we have code)
- `fix:` bug fixes
- `docs:` documentation changes
- `chore:` repo maintenance
- `data:` data files (tickets/rules/toasts)
- `tools:` scripts & tooling

Examples:
- `docs: add shift 01 draft`
- `data: add ticket templates for shift 02`
- `tools: add json validator`

## Pull Request Checklist
Before opening a PR, please verify:
- [ ] My change is focused (one topic)
- [ ] Files are placed in the correct folder
- [ ] Markdown renders correctly (headings, links, code fences)
- [ ] No references to real brands/teams/people (project policy)
- [ ] Any text meant for UI is short and readable
- [ ] I described what changed and why
